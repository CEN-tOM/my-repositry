gitの基礎

Addcommitする前に「ｽﾃｰｼﾞ」という箱に一次保存する事。

commitｵﾝﾗｲﾝﾘﾎﾟｼﾞﾄﾘに送信する前のﾛｰｶﾙﾘﾎﾟｼﾞﾄﾘに保存する事。

pushﾛｰｶﾙﾘﾎﾟｼﾞﾄﾘからｵﾝﾗｲﾝﾘﾎﾟｼﾞﾄﾘに送信する事。

    gitの手順

  1. "git init"で最初の初期手順→該当フォルダ内にﾛｰｶﾙﾘﾎﾟｼﾞﾄﾘ作成。
  2. "git status"gitの状態確認。
  3. "git add ."でフォルダ内の全てのファイルをｽﾃｰｼﾞ
  4. "git add 任意の項目"でフォルダ内の任意の項目をｽﾃｰｼﾞ
  5. "git commit -m "任意コメント""でﾛｰｶﾙﾘﾎﾟｼﾞﾄﾘに保存
    この任意コメントで何を修正追加したのかなど簡単に見出しする。
  6. "git log"ｺﾐｯﾄ履歴見れる。
  7. "git branch"でブランチ名を確認する。→確認したところ"main"だったのでOK。
    "main"ではなく"master"になっていたら"git branch -M main"と入力。
    これでブランチ名が"main"に変わる。
  8. その後"git remote add origin「githubURL」" でリモートリポジトリと接続される。
  9. 最後に"git push -u origin main"と入力する。

    git branchについて

  1. "git branch"ﾌﾞﾗﾝﾁの確認。
  2. "git branch 〇〇"〇〇という名前のﾌﾞﾗﾝﾁ作成。
  3. "git switch 〇〇"で〇〇というﾌﾞﾗﾝﾁに移動する。
  4. "git switch -c 〇〇"で〇〇というﾌﾞﾗﾝﾁ作成し移動する。
  5. "git switch -m 〇〇 △△"で〇〇というﾌﾞﾗﾝﾁを△△という名前に変更。
  6. "git branch -vv"そのﾌﾞﾗﾝﾁの上流見れる。
  7. "git push -u origin 「分岐したbranch名」"と入力するとｻﾌﾞﾌﾞﾗﾝﾁもﾘﾓｰﾄﾘﾎﾟｼﾞﾄﾘに保存される。
  8. "git branch -d 「branch名」"でﾌﾞﾗﾝﾁ削除。

My Rule

「git」や「code」などそれぞれのディレクトリ内のファイルを編集するときは、
編集前に必ずそれぞれのディレクトリに対応したbranchに移動してから編集する。
もし違うbranchで編集を進めてしまっていたら、「git stash」コマンド入力してから「git switch」
「git stash」については後ほど詳細記載する。

ローカルでマージしたあと、pushするときに下記のようなエラー出現。

このエラーが出たあとの対処法は、
"git fetch"してenter。

    リモートリポジトリからダウンロード"git clone"について

  1. リモートリポジトリから全部ダウンロードする方法。
    vscodeで新しいﾌｫﾙﾀﾞ開き、"git clone 「URL」"
    「URL」にリモートリポジトリURL貼り付けたら全部ダウンロードできる。
    ただしメインブランチしかダウンロードされない。
  2. メインブランチ以外もダウンロードしたい場合。
    "git clone -b 「ブランチ名」「URL」"でいける。
    ただし先にメインブランチをダウンロードしているのでフォルダが重複する。
    対処方として、"git clone -b 「ブランチ名」「URL」 「任意のフォルダ名」"
    これで最初にダウンロードしたメインブランチフォルダと別フォルダができる。
